# Handoff: Chromie 交互与个性化更新

**时间:** 2026-06-05  
**主程序:** `colortool.py`  
**打包产物:** `release/Chromie.exe`  
**依赖:** `Pillow`, `tkinterdnd2`  
**配置目录:** `%APPDATA%\Chromie`

## 本次更新

- 将图片导入整合为三合一区域：
  - 点击选择图片。
  - 拖拽图片到预览区。
  - 主面板激活时 `Ctrl+V` 粘贴剪贴板截图。
- 收纳为悬浮球时不响应 `Ctrl+V` 粘贴。
- 优化预览图圆角裁剪，使用高倍抗锯齿 alpha mask。
- 修复窄窗口下预览图右侧被裁切的问题。
- 预览图按图片比例自适应高度，保持完整显示，不裁切四边。
- 空态上传框使用圆角虚线边框，并跟随主题 hover 色。
- 移除旧的“导入图片 / 粘贴图片”按钮行。
- 采样色板移除重复的“复制色板”按钮，改为点击色块或文字复制当前格式值。
- 新增设置面板，默认隐藏，位于标题下方展开。
- 设置面板支持：
  - 修改标题文字，默认 `Chromie`。
  - 上传正方形 PNG 自定义头像。
  - 恢复默认头像。
  - 切换 8 套配色主题。
- 自定义头像会同时替换标题头像和悬浮球头像，并自动缩放。
- 用户配置持久化到 `%APPDATA%\Chromie\config.json`。
- 自定义头像保存到 `%APPDATA%\Chromie\avatar.png`。
- 标题、状态文字和预览文件名使用像素宽度测量后加省略号，支持多语言文本。
- 标题右侧按钮改为圆角正方形图标按钮：
  - 设置：六边形/中心点图标。
  - 收纳：窗口收纳线稿图标。
- 下拉菜单和右侧滚动条已接入主题色。
- 灰色主题改为完全无饱和度的中性 Gray。
- 修复自定义头像在悬浮球中透明裁切失效的问题：悬浮球头像现在会渲染到 `pet_key` 透明色背景，确保 Windows `transparentcolor` 能稳定裁切。

## 打包环境

本次继续使用项目内 `.build-venv` 打包。该环境基于 UE 5.6 自带 Python，Tk/Pillow/tkinterdnd2 可正常运行。

依赖安装：

```powershell
.build-venv\Scripts\python.exe -m pip install -r requirements.txt pyinstaller
```

打包命令：

```powershell
.build-venv\Scripts\pyinstaller.exe --noconfirm --clean --onefile --windowed --name Chromie --icon "D:\Asset\OtherTools\assets\chromie.ico" --add-data "D:\Asset\OtherTools\assets\chromie.png;assets" --distpath release --workpath build --specpath build colortool.py
```
